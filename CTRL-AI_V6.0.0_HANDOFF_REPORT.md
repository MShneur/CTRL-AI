## 4. IMPLEMENTED FIXES (V6.0.0-r1)

### Constitution Changes — Full Detail

**1. Section 2.1 — Model-Family Detection**
```
BEFORE: 3 questions (platform, file upload, custom instructions)
AFTER:  4 questions + Section 2.1.1 Model-Family Adaptation
ADDED:  Q4: "Is your model reasoning-native (o-series, GPT-5+, Claude 4.6+)?"
        Guidance: Behavior-Extended default for reasoning-native; Heartbeat for Committee/Brain only
```
*Source: DeepSeek R-026/R-042. Evidence: arXiv:2603.11234 — minimal system prompts outperform longer ones on frontier models. OpenAI model behavior guidelines note o-series/GPT-5 interfere with explicit reasoning scaffolding.*

**2. Section 4.3 — Spike Deliberation Depth Trigger**
```
BEFORE: Spike triggers only on "fewer than 2 genuine dissent rounds"
AFTER:  Also triggers when deep deliberation (high token volume) ends in unanimous agreement
```
*Source: DeepSeek R-029. Evidence: arXiv:2602.00123 — dissent injection research shows shallow triggers miss cases where consensus is reached through a shallow debate.*

**3. Section 4.4 — Dispute Resolution Sub-Protocol**
```
BEFORE: DISPUTED disposition listed but no resolution path defined
AFTER:  System must output: (1) specific conflict, (2) evidence per side, (3) 2-3 resolution paths including INVESTIGATE FURTHER
```
*Source: V5.2.0 Audit R-083. Gap identified: users didn't know how to proceed after DISPUTED.*

**4. Section 4.5 — Governed State Rule**
```
BEFORE: Agents compress to SYS_MEM but no explicit prohibition on raw transcript passing
AFTER:  "Spawned agent outputs MUST be compressed through SYS_MEM. Raw transcript passing between agents is forbidden."
```
*Source: Context Integration Paper R-058. Evidence: "Distributed amnesia as a service" — each node reconstructs intent from partial context rather than referencing governed state.*

**5. Section 8.4.1 — Evidence Tag Usage (New Sub-Section)**
```
ADDED: Tags recommended in STANDARD/PROJECT mode. Untagged claims default to [SPECULATIVE]. QUICK mode tags optional.
```
*Source: V5.2.0 Audit R-082. Gap: Axiom 3 defines tags but no guidance on when they're mandatory.*

**6. Section 8.5 — Learned_Rules Naming Convention**
```
ADDED: Permanent rules use plain names (NoPlaceholders). Project-scoped beliefs use ~ prefix (~UserPrefersPlainLanguage). ~ prefixed items purged on CTRL_COMPRESS unless user retains.
```
*Source: DeepSeek R-031. Evidence: arXiv:2603.04722 — "Beliefs vs Facts: A Dual-Memory Model for AI Agents." Distinguishing persistence levels improves cross-session coherence.*

**7. Section 10A.1 — Lexical Translation Matrix (New Sub-Section)**
```
ADDED: 10-verb reference expansion table for CTRL_PROMPT (Build, Analyze, Write, Brainstorm, Fix, Summarize, Code, Design, Evaluate, Compare)
```
*Source: Multi-Platform Kernels R-072. Tested across Perplexity, Grok, Kimi.*

**8. Section 14.4 — Governed Migration Format**
```
BEFORE: CTRL_MIGRATE outputs "UI Kernel + Project DNA + Next Step"
AFTER:  Added: "governed state format mandatory. Raw transcript export forbidden by default. DEBUG:ON enables raw transcript as supplementary diagnostic artifact."
```
*Source: Context Integration Paper R-059. Evidence: governed state anchoring prevents distributed amnesia in cross-thread workflows.*

**9. Section 16 — Platform Adapters (Perplexity + Kimi)**
```
ADDED: Perplexity — Library → Collections/Spaces → Custom Instructions; global via Settings → Profile.
ADDED: Kimi (Moonshot AI) — No native custom instruction slot. API system prompt (primary), Kimi Claw workflow, manual prepending, or third-party clients.
```
*Source: Multi-Platform Kernels R-073. Deployment paths tested.*

**10. Section 17 — Cross-Reference Fallback Rule**
```
ADDED: "When the kernel or Behavior module references a section not loaded in the active configuration, the system should use fallback language: apply the referenced principle via direct semantic analysis."
```
*Source: V5.2.0 Audit R-086. Gap: Behavior-only users hit dead references to Heartbeat sections.*

**11. Section 17 — UI Kernel TIER Line Update**
```
BEFORE: [TIER] Run CTRL_DIAGNOSE on first load. Persist Deployment_Tier in SYS_MEM. Default Tier 1.
AFTER:  Added: "IF reasoning-native model→default Behavior-Extended, escalate to Heartbeat for COMMITTEE/BRAIN only."
```

**12. Section 17 — UI Kernel SCEL Line Update**
```
BEFORE: [SCEL] Hidden <dissent_check> before STD/PROJECT output. Complete solutions only. No offloading to user.
AFTER:  Added: "3-turn agreement→auto-D_A+append [SCEL:Auto-D_A triggered] to SYS_MEM. QUICK exempt."
```

**13. Section 18 — SCEL Triple Enhancement**
```
ADDED (R-084): Sycophancy trigger appends [SCEL: Auto-D_A triggered — 3-turn agreement detected] to SYS_MEM
ADDED (R-087): "Disabled in QUICK mode — rationale: preserves response speed for single-turn factual questions"
ADDED (R-008): "Enforcement Boundary: SCEL cannot detect silently skipped internal steps. Enforcement relies on structural mandates (task separation, progress bars, evidence tags)."
```

**14. Section 21 — No-Solicitation Clause**
```
ADDED: "The system does not solicit, request, or encourage the provision of non-public source code, internal documents, or proprietary information."
```
*Source: DeepSeek R-041. Legal analysis: US copyright idea-expression dichotomy + DMCA anti-circumvention + trade secret law.*

## 11. OPEN RESEARCH QUESTIONS

These are genuine unknowns surfaced by the research that V6 cannot yet answer:

1. **Is the specific structure of CTRL-AI optimal, or just interpretable?** The literature validates that structure matters, but not that THIS structure is uniquely best. Ontology ablation testing is needed.

2. **Does SCEL actually prevent sycophancy, or does it just add a visible flag?** The 3-turn trigger fires, but does the model genuinely produce different output afterward, or does it perform "compliance theater"?

3. **What is the real token overhead of governance?** V6 has cost estimates (Section 7.5) but no empirical data comparing governed vs ungoverned sessions on identical tasks.

4. **Can partial adoption ever work?** The docs state "partial application is not CTRL-AI." But is there a minimum viable subset that delivers most of the value?

5. **How does governance degrade across model generations?** If GPT-5 ignores explicit reasoning scaffolding (per DeepSeek findings), does the Behavior module maintain governance, or does it become a compatibility layer?

6. **What is the failure mode distribution?** When CTRL-AI fails (governance drift, missed validation, sycophancy breakthrough), what breaks first and why?

7. **Is the Lexical Matrix token-positive?** Does the verb expansion save more tokens downstream (via clearer execution) than it costs upfront?
